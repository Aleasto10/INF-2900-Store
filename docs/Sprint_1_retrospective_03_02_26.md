# Dependency issues
    During the duration of the sprint we realized there were issues developing tasks due to 
    dependencies between tasks. We did not take enough consideration into what tasks were dependendent on others, and did not plan properly to get those tasks done early leading to a lot of work at the very end of the sprint.

# framework issues
    tasks often took more time than expected due to the fact that the framework was new to us, leading to a harsh learning curve in order to implement the tasks that we were given, as well as implementations not being as complete as expected.

# sprint goal versus end result
    while the goal of the sprint was to implement a structure and a framework to implement a minimal viable product, the end result of the sprint show issues with the structure, formatting and approach used for implementations, leading to potential difficulties in further expansion upon the code as well as potential issues in cross reviewing problems with the implementation.

# communication
    communcation between the team outside of the meeting was very sporadic and at times lacking, leading to long times between feedback and potential solutions to to problems that rose up. more meetings are needed, with thursdays being the most sought after. towards the end of the sprint communication was much better, where responsen came quick and discussions flowed freely.
